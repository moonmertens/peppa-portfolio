# Plan: Chunk 5 — About, CV & Contact Pages

## Summary

Build three new Next.js App Router pages — About, CV, and Contact — that fetch content from Sanity, apply the warm editorial design system already established in the codebase, and give the artist CMS-controlled copy with a working Web3Forms contact form.

---

## Specifications

### Functional Requirements

**About Page**
- Fetch the `aboutPage` Sanity singleton (heading, bio portable-text, artistPhoto with hotspot/crop).
- Render a two-column desktop layout: artist photo on the left, heading + bio on the right.
- On mobile the photo stacks above the text.
- Bio renders via `@portabletext/react` (already installed).
- Artist photo uses `next/image` with hotspot/crop translated to `object-position` via the existing `urlFor` helper.
- Entire page wraps sections in `FadeIn` for scroll animation.
- Exports `generateMetadata` for SEO.

**CV Page**
- Fetch all `cvEntry` documents, ordered by `year` descending.
- Group entries client-side (or at query time) into four sections with a fixed display order: Exhibitions, Education, Awards, Press.
- Each entry row shows: year (prominent), title (bold), venue/institution (muted), optional description (muted, smaller).
- Sections with zero entries are omitted entirely.
- Exports `generateMetadata` for SEO.

**Contact Page**
- Server component shell that passes `contactFormHeading` (from `siteSettings`) down to a client `ContactForm` component.
- `ContactForm` is a `'use client'` component.
- Reads `?subject=` query param via `useSearchParams` and pre-fills the subject field.
- Fields: Name (required), Email (required, email format), Subject (optional pre-fill), Message (required).
- Submits via `fetch` POST to `https://api.web3forms.com/submit` with JSON body containing `access_key`, `name`, `email`, `subject`, `message`.
- Web3Forms access key read from `process.env.NEXT_PUBLIC_WEB3FORMS_ACCESS_KEY`.
- Shows inline validation errors before submit (required fields, email regex).
- Shows a full-form success state after 2xx response.
- Shows a full-form error state on network failure or non-2xx.
- Exports `generateMetadata` for SEO.

### Non-Functional Requirements

- Server components fetch Sanity data at build/request time using the existing `sanityFetch` helper with `cache: "force-cache"`.
- No new npm packages required — all dependencies (`@portabletext/react`, `next/image`, existing fetch API) are already present.
- `NEXT_PUBLIC_WEB3FORMS_ACCESS_KEY` must be set in `.env.local` and in the Vercel project environment variables.
- Graceful fallback: if Sanity fetch fails, About page shows a minimal placeholder; CV page shows an empty state; Contact page still renders the form (form submission does not depend on Sanity).
- All pages pass basic WCAG 2.1 AA: labels on all form inputs, sufficient color contrast, keyboard-navigable.
- Consistent max-width (`max-w-5xl mx-auto px-4`) matching existing pages.
- Typography and color tokens match existing globals: `--color-cream`, `--color-ink`, `--color-warm-gray`, `--color-warm-accent`, `--color-muted`; font families `font-serif` / `font-sans`.

### Acceptance Criteria

- `/about` renders heading, photo, and bio from Sanity without errors; layout is two-column on `md:` and stacked on mobile.
- `/cv` renders at least one section heading when CV entries exist; entries are sorted newest-first within each section; empty sections are hidden.
- `/contact` form submits to Web3Forms and transitions to a success message; navigating to `/contact?subject=Some+Piece` pre-fills the subject field.
- All three pages have `<title>` tags generated by `generateMetadata`.
- TypeScript compiles with no new errors.

## Architecture

### Component Overview

| File | Type | Responsibility |
|---|---|---|
| `app/about/page.tsx` | Server component | Fetch `AboutPage` from Sanity, build photo URL, render layout, export `generateMetadata` |
| `app/cv/page.tsx` | Server component | Fetch `CvEntry[]` from Sanity, group and sort, render CV sections, export `generateMetadata` |
| `app/contact/page.tsx` | Server component (thin shell) | Fetch `contactFormHeading` from `siteSettings`, render page scaffold, mount `<ContactForm>` |
| `components/contact/ContactForm.tsx` | Client component | All form state, validation, `useSearchParams`, Web3Forms POST, success/error UI |
| `lib/types.ts` | Types | Add `AboutPage` and `CvEntry` interfaces |
| `lib/queries.ts` | GROQ queries | Add `aboutPageQuery`, `cvEntriesQuery`, extend `siteSettingsQuery` to include `contactFormHeading` |

### Data Flow

**About Page**
```
Sanity (aboutPage singleton)
  --> sanityFetch<AboutPage>(aboutPageQuery)   [server]
  --> urlFor(artistPhoto).width(900).auto('format').url()
  --> <Image> + <PortableText>
```

**CV Page**
```
Sanity (cvEntry documents)
  --> sanityFetch<CvEntry[]>(cvEntriesQuery)   [server, ordered by year desc]
  --> group by type into Map<string, CvEntry[]>
  --> render sections in fixed order
```

**Contact Page**
```
Sanity (siteSettings)
  --> sanityFetch<{ contactFormHeading }>(siteSettingsQuery)   [server]
  --> pass heading as prop to <ContactForm>

Browser
  --> useSearchParams() reads ?subject=
  --> user fills form
  --> fetch POST https://api.web3forms.com/submit
  --> success / error state
```

```typescript
export type CvEntryType = 'exhibition' | 'education' | 'award' | 'press'

export interface CvEntry {
  _id: string
  type: CvEntryType
  title: string
  venue?: string
  year: number
  description?: string
}
```

**Updated `SiteSettings` in `lib/types.ts`**

Add the optional field `contactFormHeading?: string` to the existing `SiteSettings` interface.

**New GROQ queries in `lib/queries.ts`**

```
aboutPageQuery:
  *[_type == "aboutPage"][0]{
    heading,
    bio,
    artistPhoto{
      ...,
      asset->{
        _id,
        _type,
        metadata { dimensions { width, height, aspectRatio } }
      }
    }
  }

cvEntriesQuery:
  *[_type == "cvEntry"] | order(year desc) {
    _id,
    type,
    title,
    venue,
    year,
    description
  }
```

`siteSettingsQuery` gains `contactFormHeading` in its projection (existing query extended, not replaced).

**ContactForm props**

```typescript
interface ContactFormProps {
  heading?: string
}
```

**Web3Forms POST body shape**

```typescript
{
  access_key: string          // from NEXT_PUBLIC_WEB3FORMS_ACCESS_KEY
  name: string
  email: string
  subject: string
  message: string
}
```

Response: `{ success: boolean, message: string }` — success is `true` on 2xx.

## Implementation Steps

1. **Extend `lib/types.ts`** — Add `AboutPage`, `CvEntry`, `CvEntryType` interfaces; add `contactFormHeading?: string` to `SiteSettings`.

2. **Extend `lib/queries.ts`**
   - Add `aboutPageQuery` fetching `heading`, `bio`, and `artistPhoto` with full asset dereference.
   - Add `cvEntriesQuery` fetching all CV entry fields ordered by `year desc`.
   - Extend `siteSettingsQuery` projection to include `contactFormHeading`.

3. **Create `app/about/page.tsx`**
   - `async` server component.
   - `generateMetadata`: fetch `[AboutPage, SiteSettings]` in parallel; build title `"About — {artistName}"`.
   - Default export: fetch `AboutPage` in a try/catch; call `urlFor(about.artistPhoto).width(900).height(1100).fit('crop').auto('format').url()` for the photo URL; pass the object-position from hotspot `x`/`y` if present (formula: `${hotspot.x * 100}% ${hotspot.y * 100}%`); render the two-column layout inside `<main>`.
   - Layout: `<section className="max-w-5xl mx-auto px-4 py-16 md:py-24">` containing a `<div className="flex flex-col md:flex-row gap-12 md:gap-20">`:
     - Left column `<div className="md:w-2/5 flex-shrink-0">`: `<Image>` with `fill` or fixed aspect-ratio wrapper; aspect ratio ~4:5 to suit portrait photography.
     - Right column `<div className="md:flex-1">`: optional `<h1>` heading in `font-serif text-4xl md:text-5xl`, then `<PortableText value={about.bio} />` in `font-sans text-base leading-relaxed`.
   - Wrap both columns in `<FadeIn>` with a small delay stagger.
   - Fallback when no Sanity data: single centered paragraph in `--color-warm-gray`.

4. **Create `app/cv/page.tsx`**
   - `async` server component.
   - `generateMetadata`: build title `"CV — {artistName}"`.
   - Default export: fetch `CvEntry[]`; define `SECTION_ORDER: CvEntryType[] = ['exhibition', 'education', 'award', 'press']` and `SECTION_LABELS` map.
   - Group entries: `const grouped = new Map<CvEntryType, CvEntry[]>()`. Iterate `SECTION_ORDER`, filter entries by type (already sorted year desc from GROQ), store.
   - Render `<main><div className="max-w-5xl mx-auto px-4 py-16 md:py-24">`:
     - Page-level `<h1 className="font-serif text-5xl md:text-6xl mb-16">CV</h1>` wrapped in `<FadeIn>`.
     - For each section with at least one entry:
       - `<FadeIn>` wrapping a `<section>` with `<h2 className="font-sans text-xs uppercase tracking-widest mb-6" style={{ color: 'var(--color-warm-accent)' }}>`.
       - A `<dl>` or `<ul>` of entries. Each entry is a `<div className="flex gap-6 py-4 border-b" style={{ borderColor: 'var(--color-muted)' }}>`:
         - Year: `<span className="font-serif text-lg w-16 flex-shrink-0" style={{ color: 'var(--color-warm-accent)' }}>`.
         - Body: `<div className="flex-1"><p className="font-sans text-base font-medium">title</p>` + optional `<p className="font-sans text-sm mt-0.5" style={{ color: 'var(--color-warm-gray)' }}>venue</p>` + optional `<p className="font-sans text-sm mt-1" style={{ color: 'var(--color-warm-gray)' }}>description</p></div>`.
   - Fallback when no entries: centered muted paragraph.

5. **Create `components/contact/ContactForm.tsx`**
   - `'use client'` directive at the top.
   - Imports: `useSearchParams` from `next/navigation`; `useState`, `useEffect`, `useRef` from `react`.
   - Props: `{ heading?: string }`.
   - State: `formData` object `{ name, email, subject, message }` all strings; `errors` object same keys; `status: 'idle' | 'submitting' | 'success' | 'error'`; `errorMessage: string`.
   - On mount via `useEffect`: read `searchParams.get('subject')` and set `formData.subject` if present. Use `useSearchParams()` at the top level of the component (Next.js App Router requirement).
   - Validation function `validate()`: checks non-empty name, non-empty message, valid email regex (`/^[^\s@]+@[^\s@]+\.[^\s@]+$/`). Returns errors object; empty object means valid.
   - `handleSubmit(e)`: `e.preventDefault()`; run `validate()`; if errors set them and return; set `status = 'submitting'`; POST `https://api.web3forms.com/submit` with `Content-Type: application/json` body `{ access_key: process.env.NEXT_PUBLIC_WEB3FORMS_ACCESS_KEY, ...formData }`; on success `status = 'success'`; on catch/non-ok `status = 'error'` with message.
   - Input styling shared class: `font-sans text-base w-full px-4 py-3 border outline-none transition-colors duration-200` with `borderColor: 'var(--color-muted)'`, `backgroundColor: 'transparent'`, focus border `--color-warm-accent`.
   - Label styling: `font-sans text-xs uppercase tracking-widest mb-2 block` in `--color-warm-gray`.
   - Submit button: `font-sans text-sm uppercase tracking-widest px-8 py-3` with `backgroundColor: 'var(--color-ink)'`, `color: 'var(--color-cream)'`; hover swaps to `--color-warm-accent` background; disabled + opacity-50 when `status === 'submitting'`.
   - Success state: replace the form with a `<div>` containing heading in `font-serif text-2xl` and a confirmation paragraph.
   - Error state: inline `<p role="alert">` above the submit button with the error message in `--color-warm-accent`.

6. **Create `app/contact/page.tsx`**
   - `async` server component.
   - `generateMetadata`: title `"Contact — {artistName}"`.
   - Fetch `SiteSettings` in try/catch; extract `contactFormHeading`.
   - Render `<main><div className="max-w-2xl mx-auto px-4 py-16 md:py-24">`:
     - `<FadeIn>` wrapping an optional page `<h1>` if no `contactFormHeading` (just "Contact").
     - `<FadeIn delay={100}>` wrapping `<ContactForm heading={contactFormHeading} />`.
   - The `ContactForm` component will show the heading as its own `<h1>` if provided, else a default "Get in touch".
   - Because `ContactForm` uses `useSearchParams`, wrap the import in a `<Suspense fallback={null}>` boundary inside the server component to satisfy Next.js App Router requirements.

7. **Update `.env.local`** — Add `NEXT_PUBLIC_WEB3FORMS_ACCESS_KEY=` with a blank value and a comment instructing the developer to fill it in.

## Risks / Open Questions

- **Web3Forms access key**: The artist needs to sign up at https://web3forms.com and create a key. Until `NEXT_PUBLIC_WEB3FORMS_ACCESS_KEY` is set, form submissions will fail silently with a 401. The error state in `ContactForm` will surface this as a user-visible error message.
- **`useSearchParams` + Suspense**: Next.js requires any client component that calls `useSearchParams()` to be wrapped in a `<Suspense>` boundary in the server-rendered tree. The contact server page must import `ContactForm` dynamically or wrap it in `<Suspense fallback={null}>` to avoid the build-time warning / runtime error.
- **About photo aspect ratio**: The schema allows any photo. The plan uses `fit('crop')` at 4:5 to give a consistent portrait frame. If the artist uploads a landscape photo it will be cropped. Document this expectation in the Sanity studio helper text (out of scope for this chunk, low risk).
- **Portable text serializers**: The bio likely contains only basic blocks (paragraphs, maybe bold/italic). No custom serializers are needed. If the artist adds links or images to the bio in future, a custom `components` prop to `<PortableText>` will be required in a follow-up.
- **Empty CV**: If no CV entries exist in Sanity yet, the page renders only the `<h1>CV</h1>` and a "No entries yet" placeholder. This is acceptable.
- **siteSettingsQuery change**: Adding `contactFormHeading` to the existing GROQ projection is backward-compatible — existing call sites destructure only what they need, so they are unaffected.
